Diagram 3 of 3 — UML Sequence

Policy Purchase — End-to-End Interaction

The complete lifecycle of a policy purchase — from JWT authentication, through CQRS-based policy creation, to Razorpay payment verification, Saga pattern event propagation via RabbitMQ, and email notification dispatch. 
sequenceDiagram
    actor Customer
    participant FE as Frontend
    participant GW as Gateway
    participant AUTH as Auth
    participant POL as Policy
    participant PAY as Payment
    participant RMQ as RabbitMQ
    participant DB as Database

    Customer->>FE: Login
    FE->>GW: POST /auth/login
    GW->>AUTH: Validate credentials
    AUTH->>DB: Query user
    AUTH-->>FE: JWT Token

    Customer->>FE: Select policy
    FE->>GW: POST /policy/buy (JWT)
    GW->>POL: Create user policy
    POL->>DB: Save user_policy (PENDING)
    POL-->>FE: userPolicyId

    FE->>GW: POST /payment/create-order
    GW->>PAY: Create Razorpay order
    PAY->>DB: Save transaction (PENDING)
    PAY-->>FE: orderId + amount

    Note over Customer, FE: Razorpay Checkout
    Customer->>FE: Complete payment
    FE->>GW: POST /payment/verify
    GW->>PAY: Verify Razorpay signature

    alt Valid
        PAY->>DB: Transaction → SUCCESS
        PAY->>RMQ: PaymentStatusEvent
        RMQ->>POL: Saga Consumer
        POL->>DB: UserPolicy → ACTIVE
        PAY-->>Customer: Confirmation email
    else Invalid
        PAY->>DB: Transaction → FAILED
        PAY->>RMQ: FAILED event
        RMQ->>POL: Saga Consumer
        POL->>DB: UserPolicy → CANCELLED
    end

    PAY-->>FE: Result
    FE-->>Customer: Display status
